# Computing Resource Workflow Example

This example pack is a local, evidence-backed report generation workflow. It exists to prevent an
agent from presenting fixture values as real resource usage.

The current implementation refreshes `evidence/local-execution.json` from the machine running the
validator, then generates `outputs/resource-usage-report.md` from that evidence. If there is no
GPU usage or billing evidence, the report must not claim GPU usage or cost.

## Inputs

- `inputs/resource-request.yaml`
- `workflow/resource-workflow.yaml`

`resource-request.yaml` fixes the command and output report contract. `resource-workflow.yaml`
keeps the five-step workflow shape while making the approval step reject unsupported resource
claims.

## Evidence

- `evidence/local-execution.json`

This file is generated by the validator. It records the local platform, Python version, logical CPU
cores, memory, `nvidia-smi` availability, whether GPU usage or cost is claimed, and the concrete
`workflow_trace` that matches `workflow/resource-workflow.yaml`.

## Outputs

- `outputs/resource-usage-report.md`
- CLI JSON summary with the local execution evidence used by the report

The report includes a `Workflow Trace` section rendered from the same evidence file. The validator
fails if the trace is missing or disagrees with the workflow YAML.

## As-Is

- An agent could previously write GPU model, GPU quantity, CPU allocation, or cost values without
  a matching evidence artifact.
- That made a local validation run look like it used external NVIDIA GPUs and billing spend.

## To-Be

- The report uses only `evidence/local-execution.json` values refreshed in the current environment.
- The workflow is proven by `workflow_trace`, not by step names alone.
- GPU usage and cost are reported as unclaimed unless a real evidence artifact exists.
- The validator fails if the report differs from the evidence-backed generated output.

## Platform Effects

- The pack is reproducible on macOS, Linux, and CI as long as Python 3.13+ and `uv` are available.
- Runtime-specific facts are refreshed into evidence before the report is generated.
- Unsupported resource claims are removed instead of being dressed up as measured results.

## Validate

```bash
uv run python scripts/validate_computing_resource_example.py examples/computing-resource-workflow --write-report
uv run python scripts/validate_computing_resource_example.py examples/computing-resource-workflow
```
